## medium: Declared commit and release targets are silent no-ops

id: fnd_sig-feat-config-0902ad7be8-24994_20da3ff139
category: build-release
confidence: high
triage: risk
status: open
feature: Project config Makefile (feat_config_0902ad7be8)

evidence:
- Makefile:1

The Makefile declares commit and release as phony targets but does not define recipes for either target anywhere in the inspected file. In GNU Make, a phony target with no recipe is still considered a valid target, so `make release` or `make commit` exits successfully while doing nothing. That is a release/build hazard because automation or a developer can receive a successful exit status for an operation that was never performed.

recommendation:
Either remove commit and release from .PHONY so unsupported invocations fail, or add real recipes for both targets and list them in help.

test analysis:
No linked tests or Makefile smoke checks exercise phony target definitions; Go tests do not validate Make target behavior.

suggested regression test:
Add a CI smoke test that fails when declared phony targets have no recipe, or specifically asserts that `make -n release` and `make -n commit` produce expected commands or fail if unsupported.

minimum fix scope:
Update Makefile target declarations and, if these targets are intended, add the missing commit/release recipes.

repro:
Run `make -n release`; it exits 0 and reports that nothing needs to be done.

## medium: EMA cache panics after the max cache size

status: open

suggested regression test:
Add an EMA test with a series longer than defaultMaxCacheSize+1 that calls Calculate(defaultMaxCacheSize+1) and asserts it does not panic and returns a deterministic value. Add a direct cacheResult or NewCache.Set boundary test for index == maxCacheSize and index > maxCacheSize.

minimum fix scope:
Update pkg/indicators/cached_indicator.go bounds handling and add boundary regression tests for cached indicators.

repro:
Create an EMA for a time series with at least 10002 candles and call Calculate(10001); the recursive calculation fills the cache to 10000, skips caching index 10000, then panics while caching index 10001.
